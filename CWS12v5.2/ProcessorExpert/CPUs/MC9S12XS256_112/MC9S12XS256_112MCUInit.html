Device Initialization
(Description of the CPU component parameters.)
  • Clock settings - MCU clock settings
    • Input clock - Settings of the input clock signals.
      • Clock type - This property determines allowed range of the input clock frequency. It should be set according to your hardware configuration.
        The allowed range for the selected clock type is shown in the third column of this property.
        This property doesn't modify any registers, it is used only for input clock range checking.
      • Clock frequency [MHz] - Frequency of the main clock. The crystal or external (square wave) clock generator can be used as a clock source. The allowed range of frequency depends on clock type and is shown in the third column of the clock type property.
      • Clock input pin - Clock input properties.
        • Pin - Clock input pin name (for information only).
      • Clock output pin - Clock output properties.
        • Pin - Clock output pin name (for information only).
    • Output clock - Settings of the output clock signals.
      • External clock - This property controls the availability of the clock on the ECLK pin. The clock output is always active in emulation modes and if it is enabled, in all other operating modes as well.
        The following items are available only if the group is enabled (the value is "Enabled"):
        • ECLK pin - External clock output pin (for information only).
        • ECLK predivider - This property controls ECLK predivider by 16. When Disabled, the ECLK clock rate is defined by setting of the ECLK clock rate property. When Enabled an additional divide-by-16 is enabled for the selected ECLK clock rate.
        • ECLK clock rate - This property determines the rate of the free-running clock on the ECLK pin. The divider is always disabled in emulation modes and it is active as it is programmed in all other operating modes.
      • Fixed external clock - This property controls the availability of the free-running clock on the ECLKX2 pin. This clock has a fixed rate of twice the internal bus clock. The clock output is always active in emulation modes and in all other operating modes if it is enabled.
        The following items are available only if the group is enabled (the value is "Enabled"):
        • ECLKX2 pin - Fixed external clock output pin (for information only).
    • Internal bus clock - The Bus clock is derived from System clock (Fsys) - Fbus = Fsys/2.
      When the PLL clock property is set to 'Disabled' value:
      Fbus = Oscillator frequency / 2
      When the PLL clock property is set to 'Enabled' value:
      Fbus = VCO clock frequency / (2 * PLL post divider)
      When properties Reference divider, PLL multiplication factor and PLL post divider are all not set to 'Auto select' value or when the PLL clock is disabled, Internal bus clock is calculated by Processor Expert and cannot be directly modified.
    • PLL clock - This property selects clock source for the System clock(Fsys).
      If set to 'Enabled' value then:
      1. System clocks are derived from the PLL clock (Fsys = Fpll).
      2. Fbus = Fsys / 2 = Fpll / 2 = 2 * Fosc * Mult/ (2 * Div * PostDiv) = Fosc * Mult/ (Div * PostDiv) .
      If set to 'Disabled' value then:
      1. System clocks are derived from the Oscillator clock (Fsys = Fosc).
      2. Fbus = Fsys / 2 = Fosc / 2

      The following items are available only if the group is enabled (the value is "Enabled"):
      • Reference divider - This property provides a finer granularity for the PLL multiplier steps. Processor Expert selects the best value for the desired bus clock and the Oscillator frequency if the 'Auto select' value is selected.
      • PLL multiplication factor - This property controls the multiplication factor of the PLL. Processor Expert selects the best value for the current bus clock and the Oscillator frequency if the 'Auto select' value is selected.
      • PLL post divider - This property controls the post programmable divider of the PLL. Processor Expert selects the best value for the current bus clock and the Oscillator frequency if the 'Auto select' value is selected.
      • VCO clock frequency [MHz] - Frequency of the Voltage Controlled Oscillator [MHz] (for information only). It is calculated by Processor Expert according to the formula:
        Fvco = 2 * Fosc / Reference divider * PLL multiplication factor
        The value of PLL output clock is derived from the VCO clock:
        Fpll = Fvco / PLL post divider
      • Frequency Modulation - This property enables additional frequency modulation on the VCO clock. This is to reduce noise emission. The modulation frequency is fref divided by16.
    • Low-power modes settings - This group controls the CRG module behavior in low-power modes.
      • Pseudo stop - This property controls the functionality of the oscillator during the stop mode.
        If the value of this property is 'yes' the oscillator continues to run in the stop mode (pseudo stop). The oscillator amplitude is reduced.
        If the value of this property is 'no' the oscillator is disabled in the stop mode.
        NOTE:
        Pseudo-STOP allows for faster STOP recovery and reduces the mechanical stress and aging of the resonator in case of frequent STOP conditions at the expense of slightly increased power consumption. Lower oscillator amplitude exhibits lower power consumption but could have adverse effects during any Electro-Magnetic Susceptibility (EMS) tests.
      • Fast wake-up from full stop mode - This property enables/disables fast wake-up from full stop mode.
        If the value of this property is 'yes' the fast wake-up from full stop mode is enabled.
        If the value of this property is 'no' the Fast wake-up from full stop mode is disabled.
      • PLL stops in wait mode - This property determines a behavior of the PLL module in the wait mode
        If the value of this property is 'yes' then the PLL module will operate in the wait mode.
        If the value of this property is 'no' then the PLL module stops in the wait mode.
    • Clock monitor - This property specifies the behavior of the clock monitor.

      There are 3 options:
      • Disabled: Clock monitor is disabled.
      • Force reset: If the loss of oscillator/external clock is detected the CRG generates a clock monitor fail reset.
      • Enter Self clock mode: If the loss of oscillator/external clock is detected the CRG generates SCM interrupt and enters Self-clock mode.

  • Interrupts initialization - Initial interrupt settings (CCR register)
    • Initialization priority - Initialization of the Interrupt Processing Level (IPL) and Enabling/Disabling I-maskable interrupts. HCS12X core has seven priority levels.
      Possible values are:
      • minimal priority - Enables interrupts with minimal and higher priority (1,2,3,4,5,6,7)
      • low priority - Enables interrupts with low and higher priority (3,4,5,6,7)
      • medium priority - Enables interrupts with medium and higher priority (4,5,6,7)
      • high priority - Enables interrupts with high and higher priority (5,6,7)
      • maximal priority - Enables interrupts with maximal priority (7)
      • interrupts enabled - Interrupts enabled, IPL not modified, default value
      • interrupts disabled - Interrupts disabled, IPL not modified
      • 0 - Interrupts enabled, IPL=0 - Enables interrupts of level 1,2,3,4,5,6,7 (all interrupts),
      • 1 - Interrupts enabled, IPL=1 - Enables interrupts of level 2,3,4,5,6,7
      • 2 - Interrupts enabled, IPL=2 - Enables interrupts of level 3,4,5,6,7
      • 3 - Interrupts enabled, IPL=3 - Enables interrupts of level 4,5,6,7
      • 4 - Interrupts enabled, IPL=4 - Enables interrupts of level 5,6,7
      • 5 - Interrupts enabled, IPL=5 - Enables interrupts of level 6,7
      • 6 - Interrupts enabled, IPL=6 - Enables interrupts of level 7
      • 7 - Interrupts enabled, IPL=7 - Only system interrupts are allowed, I-maskable interrupts are disabled by the priority of value 7
  • Internal resource mapping - Internal resource mapping setting
    • Direct page mapping - This property determines the position of the direct page within the memory map. The CPU when performing accesses using the direct addressing mode uses this value. The value represents the upper eight bits [15:8] of the direct page address.
    • Interrupt/Reset vector table - This group contains the vector table placement settings.
      • Reset vector table - This group contains settings for placement of reset vectors.
        • Address - Address of the reset vectors part of the interrupt vectors table (IVT). Reset vector part of IVT will be placed on this address. The value doesn't affect any HW resources and it is intended for a bootloader or monitor support. This value only affects the address of the reset vectors part of the interrupt vector table in the Vectors.c file. The default address is 0xFFFA.
        • Size - Size of the reset vectors part of the interrupt vector table. (For information only)
      • Interrupt vector table - This group contains settings for placement of interrupt vectors.
        • Address - Address of the interrupt vector table (IVT). IVT will be placed on this address. This value affect IVBR register, but setting of this register can be suppressed (If the property 'Init IVBR reg.' is set to 'no') for bootloader or monitor support. The default address is 0xFF10.
        • Size - Size of the interrupt vector table. (For information only)
        • Init IVBR register - Initialization of IVBR (Interrupt Vector Base Register) can be suppressed by setting this property to 'no' value. This feature can be useful for bootloader or monitor support. If enabled, IVBR is initialized with high byte value of interrupt vector table address. Note: Low byte of interrupt vector table should be set to 0x10.

    • Operating mode switching - This property enables to generate an operating mode switching code, i.e. it enables changing of the boot operating mode by application code. This property allows user to switch from the boot operating mode into the selected operation mode. If the code for switching of operating mode is undesirable, be aware that most debugging tools provide the ability to self configure the device in the debugging environment (if the part is booted into special signal-chip mode). If the CPU is to be configured for expanded operation in secure mode, the CPU must exit reset in the expanded mode. No writes to the MOD bits are allowed while operating in the secure mode. However, to release security, special single-chip mode must be possible. For more information see the Application note AN2287

  • Internal peripherals - Internal peripherals setting
    • BDM Debug support - Setting for BDM debugging.
      • Stop COP and RTI in Active BDM mode - If the value of this property is 'yes' the COP and RTI module counters are stopped in Active BDM mode.
        Derivatives with CRG or ECRG module: This item modifies the RSBCK bit in the COPCTL register.
        Derivatives with CPMU module: This item modifies the RSBCK bit in the CPMUCOP register.
    • I/O module - I/O ports' settings
      • Initialize unused I/O pins - Initialization for all unused I/O pins ( I/O pins neither used by any component nor reserved in the 'Used' tab of the cpu component).
        This property specifies initialization direction for all unused pins . If the given unused pin doesn't support the preferred direction then the pin direction is left in it's default state. Note: Initial state of the unused pins is displayed in Target CPU package window.
        There are 3 modes:
        • no initialization - There are no items in this mode.
        • input - The following items are displayed in this mode:
          • Pull resistor - Default initialization of pull resistors for all unused I/O pins. Pull resistor is initialized only if the given unused pin is set to input direction and supports selected pull resistor type. Pull resistor initialization is not done for ports with pull setting common for all pins (A, B, C, D, E and K).
          • Output value - Default initialization of output value for all unused I/O pins. Output value is initialized only if the given unused pin is set to output direction.
        • output - See input mode for items available in this mode.
      • PORT A - Reduced drive setting for pins of PORT A
      • PORT AD0H - Reduced drive setting for pins of PORT AD0H
      • PORT AD0L - Reduced drive setting for pins of PORT AD0L

      • PORT K - Reduced drive setting for pins of PORT K
