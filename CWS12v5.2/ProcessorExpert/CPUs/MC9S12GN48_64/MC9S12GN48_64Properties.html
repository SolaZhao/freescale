CPU Component MC9S12GN48_64
Freescale HC9S12 family: MC9S12GN48 in 64 pin package
Component Level: High
Category: CPU-Freescale-HCS12-G
Properties:
(Properties are parameters of the component. Please see the Embedded Components page for more information.)
  • Component name - Name of the component
  • CPU type - Cpu variant
  • Clock settings - MCU clock settings
    • Internal clock - Internal clock settings.
      • Internal oscillator frequency [kHz] - Frequency of the internal oscillator. Factory trimmed to value 1MHz.
    • External clock - If this property is enabled, an external clock is allowed for MCU timing.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • Clock frequency [MHz] - Frequency of the main clock. The crystal or external (square wave) clock generator can be used as a clock source.
      • Clock input pin - Clock input properties.
        • Pin - Clock input pin name (for information only).
        • Pin signal - Clock input signal name.
      • Clock output pin - Clock output properties.
        • Pin - Clock output pin name (for information only).
        • Pin signal - Clock output signal name.
    • Output clock - Settings of the output clock signals.
      • ECLK clock - This property controls the availability of the clock on the ECLK pin.
        The following items are available only if the group is enabled (the value is "Enabled"):
        • ECLK pin - External clock output pin (for information only).
        • ECLK pin signal - Signal name of ECLK pin.
        • ECLK predivider - This property controls ECLK predivider by 16. When Disabled, the ECLK clock rate is defined by setting of the ECLK clock rate property. When Enabled an additional divide-by-16 is enabled for the selected ECLK clock rate.
        • ECLK clock rate - This property determines the rate of the free-running clock on the ECLK pin. The divider is always disabled in emulation modes and it is active as it is programmed in all other operating modes.
      • ECLKX2 clock - This property controls the availability of the free-running clock on the ECLKX2 pin. This clock has a fixed rate of twice the internal bus clock. The clock output is always active in emulation modes and in all other operating modes if it is enabled.
        The following items are available only if the group is enabled (the value is "Enabled"):
        • ECLKX2 pin - Fixed external clock output pin (for information only).
        • ECLKX2 pin signal - Signal name of ECLKX2 pin.
    • Low-power modes settings - This group controls the CPMU module behavior in low-power modes.
      • Pseudo stop - This property controls the functionality of the oscillator during the stop mode.
        If the value of this property is 'yes' the oscillator continues to run in the stop mode (pseudo stop). The oscillator amplitude is reduced.
        If the value of this property is 'no' the oscillator is disabled in the stop mode.
        NOTE:
        Pseudo-STOP allows for faster STOP recovery and reduces the mechanical stress and aging of the resonator in case of frequent STOP conditions at the expense of slightly increased power consumption. Lower oscillator amplitude exhibits lower power consumption but could have adverse effects during any Electro-Magnetic Susceptibility (EMS) tests.
  • Initialization priority - Initialization interrupt priority value
  • Internal resource mapping - Internal resource mapping setting
    • Direct page mapping - This property determines the position of the direct page within the memory map. The CPU when performing accesses using the direct addressing mode uses this value. The value represents the upper eight bits [15:8] of the direct page address.
    • NVM resources mapping disabled - If yes, Program Flash is mapped to global addresses 0x4000 - 0x7FFF. If no, NVM resources are mapped to global address range 0x4000 - 0x7FFF.
    • Interrupt/Reset vector table - This group contains the vector table placement settings.
      • Reset vector table - This group contains settings for placement of reset vectors.
        • Address - Address of the reset vectors part of the interrupt vectors table (IVT). Reset vector part of IVT will be placed on this address. The value doesn't affect any HW resources and it is intended for a bootloader or monitor support. This value only affects the address of the reset vectors part of the interrupt vector table in the Vectors.c file. The default address is 0xFFFA.
        • Size - Size of the reset vectors part of the interrupt vector table. (For information only)
      • Interrupt vector table - This group contains settings for placement of interrupt vectors.
        • Address - Address of the interrupt vector table (IVT). IVT will be placed on this address. This value affect IVBR register, but setting of this register can be suppressed (If the property 'Init IVBR reg.' is set to 'no') for bootloader or monitor support. The default address is 0xFF10.
        • Size - Size of the interrupt vector table. (For information only)
        • Init IVBR register - Initialization of IVBR (Interrupt Vector Base Register) can be suppressed by setting this property to 'no' value. This feature can be useful for bootloader or monitor support. If enabled, IVBR is initialized with high byte value of interrupt vector table address. Note: Low byte of interrupt vector table should be set to 0x10.
  • Operating mode settings - CPU operating mode settings.
    • Boot operating mode - This property must be set according to the CPU operating mode that is set after reset (respectively before execution of an application). This MCU mode is set by PE5/MODA, PE6/MODB and BKGD/MODC pins during reset (respectively set by debugger).
      Note:
      Processor Expert needs to know an operating mode to allocate IO pins required for selected mode and make some settings available.

      There are 2 options:
      • Special Single Chip: A debug mode which causes the device to enter BDM Active Mode after each reset
      • Normal Single Chip: The mode used for running applications

    • Operating mode switching - This property enables to generate an operating mode switching code, i.e. it enables changing of the boot operating mode by application code. This operating mode is set by the Target operation mode property (available only if this property is set to 'yes') Note: Sometimes it is required to boot into some operation mode and switch to another mode by writing to the MODE register. This property allows user to switch from the boot operating mode into the selected operation mode. If the code for switching of operating mode is undesirable, be aware that most debugging tools provide the ability to self configure the device in the debugging environment (if the part is booted into special signal-chip mode). If the CPU is to be configured for expanded operation in secure mode, the CPU must exit reset in the expanded mode. No writes to the MOD bits are allowed while operating in the secure mode. However, to release security, special single-chip mode must be possible. For more information see the Application note AN2287
      The following items are available only if the group is enabled (the value is "yes"):
      • Target operation mode - This property defines operation mode set by the generated code by writing to the MODC bit of the MODE register. This property is available only if the value of the Operating mode switching property is set to 'yes'

        There are 2 options:
        • Special Single Chip: A debug mode which causes the device to enter BDM Active Mode after each reset
        • Normal Single Chip: The mode used for running applications

  • Internal peripherals - Internal peripherals setting
    • BDM Debug support - Setting for BDM debugging.
      • Stop COP and RTI in Active BDM mode - If the value of this property is 'yes' the COP and RTI module counters are stopped in Active BDM mode.
        Derivatives with CRG or ECRG module: This item modifies the RSBCK bit in the COPCTL register.
        Derivatives with CPMU module: This item modifies the RSBCK bit in the CPMUCOP register.
    • I/O module - I/O ports' settings
      • Initialize unused I/O pins - Initialization for all unused I/O pins ( I/O pins neither used by any component nor reserved in the 'Used' tab of the cpu component).
        This property specifies initialization direction for all unused pins . If the given unused pin doesn't support the preferred direction then the pin direction is left in it's default state. Note: Initial state of the unused pins is displayed in Target CPU package window.
        There are 3 modes:
        • no initialization - There are no items in this mode.
        • input - The following items are displayed in this mode:
          • Pull resistor - Default initialization of pull resistors for all unused I/O pins. Pull resistor is initialized only if the given unused pin is set to input direction and supports selected pull resistor type. Pull resistor initialization is not done for ports with pull setting common for all pins (A, B, C, D, E and K).
          • Output value - Default initialization of output value for all unused I/O pins. Output value is initialized only if the given unused pin is set to output direction.
        • output - See input mode for items available in this mode.
    • PWM module - This is a global setting specific for the CPU component, which is not visible in this component but may be mirrored into another components.
      • Stop in wait mode - If this property is 'yes', it allows for lower power consumption in wait mode by disabling the input clock to the prescaler. This item modifies PSWAI bit in PWMCTL register.
      • Stop in freeze mode - In Freeze Mode, there is an option to disable the input clock to the prescaler by setting this property to 'yes'. If this property is 'yes' then whenever the MCU is in freeze mode the input clock to the prescaler is disabled. This item modifies PFRZ bit in PWMCTL register.
      • Emergency shutdown feature - Enables Emergency shutdown feature of PWM periphery and opens other settings up. This item modifies PWM5ENA bit in PWMSDN register.
      • Shutdown pin settings - If this property is "Enabled" the Emergency shutdown input pin of the PWM module is allocated and other settings are available.
        The following items are available only if the group is enabled (the value is "Enabled"):
        • Input pin - Input pin used for Emergency shutdown feature.
        • Input active level - This property determines the active level of the Emergency shutdown input pin. This item modifies PWM5INL bit in PWMSDN register.
        • Shutdown output level - If active level, as defined by the previous property, gets asserted all enabled PWM channels are immediately driven to the level defined by this property. This item modifies bit PWMLVL in PWMSDN register.
    • TIM module - This is a global setting specific for the CPU component, which is not visible in this component but may be mirrored into another components.
      • Stop in wait mode - Disable the timer module when the MCU is in the wait mode. Timer interrupts cannot be used to get the MCU out of wait.
        Pulse accumulator is also affected.
      • Stop in freeze mode - Disable the timer counter whenever the MCU is in freeze mode.
        Pulse accumulator will not stop.
    • FLASH - The FLASH memory options.
      • Security state - If the security state is enabled the content of the memory cannot be read by an instruction from any unsecured source including the background debug interface.
        When the value is set to 'Not initialized', the NVSEC register is not written. When the value is euther 'Enabled' or 'Disabled', the NVSEC register is initialized accordingly.
        There are 3 modes:
        • Not initialized - There are no items in this mode.
        • Enabled - The following items are displayed in this mode:
          • Backdoor key mechanism - This property enables/disables security unlocking mechanism. This mechanism uses an 8-byte backdoor security key to unsecure the MCU. You can use SetBackdoorKey() method to unsecure the MCU (Not available in Device Initialization tool).
            The following items are available only if the group is enabled (the value is "Enabled"):
            • Backdoor key 0 - 7 Backdoor key value of the register 0-7.
        • Disabled - There are no items in this mode.
  • CPU interrupts/resets - Interrupts/Resets vectors allocated by the CPU component.
    • Clock monitor reset - This property enables/disables generation of the 'Clock monitor reset' reset vector routine. In case the clock module allows to disable the clock monitor (there is a 'Clock monitor' property in the 'Clock settings' group), this property is enabled only if the property 'Clock monitor' is set to 'Force reset' value.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • Interrupt - Interrupt associated with clock monitor.
    • IllegalOpcode - This property enables/disables generation of the 'Illegal operation code' interrupt service routine.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • Interrupt - Interrupt associated with illegal opcode detection.
    • SWI - This property enables/disables generation of SWI interrupt service routine.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • Interrupt - Interrupt associated with the SWI.
    • LVD - This property enables/disables generation of the 'LVD status change' interrupt and its ISR.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • Interrupt - Interrupt associated with the LVD.
      • Interrupt priority - Interrupt priority of LVD interrupt.
    • PLL - This property enables/disables generation of the 'PLL LOCK status change' interrupt and its ISR. Settings of this property is enabled only if the property PLL clock is set to 'Enabled' value.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • Interrupt - Interrupt associated with the PLL.
      • Interrupt priority - Interrupt priority of PLL interrupt.
    • Oscillator Status Changed - This property enables/disables generation of the 'Oscillator Status Changed' interrupt and its ISR.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • Interrupt - Interrupt associated with Oscillator Status Changed.
      • Interrupt priority - Interrupt priority of Oscillator Status Changed interrupt.
    • Spurious interrupt - This property enables/disables generation of 'Spurious interrupt' interrupt service routine.
      The following items are available only if the group is enabled (the value is "Enabled"):
      • Interrupt - Interrupt associated with the spurious interrupt.
      • Interrupt priority - Interrupt priority of spurious interrupt.
  • Enabled speed modes - Enabled speed modes
    • High speed mode - High speed mode support
      The following items are available only if the group is enabled (the value is "Enabled"):
      • High speed clock - The clock source for the internal bus clock.
        When set to 'Internal clock', the internal 1MHz oscillator is used as a reference clock for PLL.
        When set to 'External clock', the external oscillator is used as a reference clock for PLL. To use the 'External clock' value, the External clock property must be enabled.
        The number in the third column displays the frequency of the source clock of Processor Expert timing model. When PLL is enabled, the value is the output frequency of the PLL clock. When PLL is disabled, the value is the frequency of the External clock.
      • Internal bus clock - The internal bus clock frequency. When the PLL clock property is set to 'Enabled', the bus clock is based on the PLL clock.
        When the PLL clock property is set to 'Disabled, the bus clock is based on the Oscillator Clock.
        When properties Reference divider, PLL multiplication factor and PLL post divider are all not set to 'Auto select' value or when the PLL clock property is disabled, Internal bus clock is calculated by Processor Expert and cannot be directly modified.
      • PLL clock - This property selects clock source for the system clocks (bus clock and core clock).
        When set to 'Enabled' value the PLL clock is used as the clock source:
        Fbus = Fpll / 2 = Fref * Mult/ (Div * PostDiv)
        Fref - reference clock can be either internal 1MHz oscillator or external oscillator.
        When set to 'Disabled' value the external oscillator clock is used as the clock source:
        Fbus = Fosc / 2
        It is not possible to use the internal 1MHz oscillator as a clock source PLL is disabled.
        The following items are available only if the group is enabled (the value is "Enabled"):
        • Reference divider - This property provides a finer granularity for the PLL multiplier steps. Processor Expert selects the best value for the desired bus clock and the selected clock source if the 'Auto select' value is selected.
        • PLL multiplication factor - This property controls the multiplication factor of the PLL. Processor Expert selects the best value for the current bus clock and the selected clock source if the 'Auto select' value is selected.
        • PLL post divider - This property controls the post programmable divider of the PLL. Processor Expert selects the best value for the current bus clock and the selected clock source if the 'Auto select' value is selected.
        • VCO clock frequency [MHz] - Frequency of the Voltage Controlled Oscillator [MHz] (for information only). It is calculated by Processor Expert according to the formula:
          Fvco = 2 * Fref / Reference divider * PLL multiplication factor
          Fref - reference clock can be either internal 1MHz oscillator or external oscillator.
          The value of PLL output clock is derived from the VCO clock:
          Fpll = Fvco / PLL post divider
        • Frequency Modulation - This property enables additional frequency modulation on the VCO clock. This is to reduce noise emission. The modulation frequency is Fref divided by16.
    • Low speed mode - See High speed mode description.
    • Slow speed mode - See High speed mode description.
