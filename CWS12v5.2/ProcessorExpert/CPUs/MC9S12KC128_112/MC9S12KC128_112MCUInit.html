Device Initialization
(Description of the CPU component parameters.)
  • Clock settings - MCU clock settings
    • Input clock - Settings of the input clock signals.
      • Clock type - This property determines allowed range of the input clock frequency. It should be set according to your hardware configuration.
        The allowed range for the selected clock type is shown in the third column of this property.
        This property doesn't modify any registers, it is used only for input clock range checking.
      • Clock frequency [MHz] - Frequency of the main clock. The crystal or external (square wave) clock generator can be used as a clock source. The allowed range of frequency depends on clock type and is shown in the third column of the clock type property.
      • Clock input pin - Clock input properties.
        • Pin - Clock input pin name (for information only).
      • Clock output pin - Clock output properties.
        • Pin - Clock output pin name (for information only).
    • Output clock - Settings of the output clock signals.
      • External clock - This property controls the availability of the clock on the ECLK pin. The clock output is always active in emulation modes and if it is enabled, in all other operating modes as well.
        The following items are available only if the group is enabled (the value is "Enabled"):
        • ECLK pin - External clock output pin (for information only).
    • Internal bus clock - Internal bus clock [MHz]. The bus clock(Fbus) is a clock source for internal on-chip peripherals.
      The Bus clock is derived from System clock (Fsys) - Fbus = Fsys/2.
      Fbus = Oscillator frequency / 2 if the PLL clock property is set to 'Disabled' value.
      Fbus = PLL clock frequency / 2 if the PLL clock property is set to 'Enabled' value.
      If both Reference divider and PLL multiplication factor are not set to 'Auto select' value or if the PLL clock is disabled, Internal bus clock is calculated by Processor Expert and cannot be directly modified.
    • PLL clock - This property select clock source for the System clock(Fsys).
      If set to 'Enabled' value then:
      1. System clocks are derived from the PLL clock (Fsys = Fpll).
      2. Fbus = Fsys / 2 = Fpll / 2 = 2 * Fosc / Div * Mult / 2 = Fosc / Div * Mult .
      If set to 'Disabled' value then:
      1. System clocks are derived from the Oscillator clock (Fsys = Fosc).
      2. Fbus = Fsys / 2 = Fosc / 2

      The following items are available only if the group is enabled (the value is "Enabled"):
      • Reference divider - This property provides a finer granularity for the PLL multiplier steps. Processor Expert selects the best value for the desired bus clock and the Oscillator frequency if the 'Auto select' value is selected.
      • PLL multiplication factor - This property controls the multiplication factor of the PLL. Processor Expert selects the best value for the current bus clock and the Oscillator frequency if the 'Auto select' value is selected.
      • PLL clock frequency - PLL clock frequency [MHz]. This property is for information only. It is calculated by Processor Expert according to the formula:
        Fpll = 2 * Fosc / Reference divider * PLL multiplication factor
      • PLL bandwidth control - Selects automatic or manual bandwidth control.
      • PLL mode - Selects acquisition or tracking mode.
    • Low-power modes settings - This group controls the CRG module behavior in low-power modes.
      • Pseudo stop - This property controls the functionality of the oscillator during the stop mode.
        If the value of this property is 'yes' the oscillator continues to run in the stop mode (pseudo stop). The oscillator amplitude is reduced.
        If the value of this property is 'no' the oscillator is disabled in the stop mode.
        NOTE:
        Pseudo-STOP allows for faster STOP recovery and reduces the mechanical stress and aging of the resonator in case of frequent STOP conditions at the expense of slightly increased power consumption. Lower oscillator amplitude exhibits lower power consumption but could have adverse effects during any Electro-Magnetic Susceptibility (EMS) tests.
      • System clocks stop in wait mode - This property controls the functionality of the oscillator during the wait mode.
        If the value of this property is 'yes' then the system clocks are stopped in the wait mode.
        If the value of this property is 'no' then the system clocks continue to run in the wait mode.
      • Reduced oscillator amplitude in wait mode - This property determines an oscillator amplitude in the wait mode
        If the value of this property is 'yes' the oscillator amplitude will be reduced in the wait mode.
        If the value of this property is 'no' there will be normal oscillator amplitude in the wait mode.
      • Core stops in wait mode - This property determines a behavior of the core clock in the wait mode
        If the value of this property is 'yes' then the core clock stops in the wait mode.
        If the value of this property is 'no' then the core clock keeps running in the wait mode.
      • PLL stops in wait mode - This property determines a behavior of the PLL module in the wait mode
        If the value of this property is 'yes' then the PLL module will operate in the wait mode.
        If the value of this property is 'no' then the PLL module stops in the wait mode.
    • Clock monitor - This property specifies the behavior of the clock monitor.

      There are 3 options:
      • Disabled: Clock monitor is disabled.
      • Force reset: If the loss of oscillator/external clock is detected the CRG generates a clock monitor fail reset.
      • Enter Self clock mode: If the loss of oscillator/external clock is detected the CRG generates SCM interrupt and enters Self-clock mode.

  • Internal resource mapping - Internal resource mapping setting
    • Register block mapping - Register block mapping address.
    • Internal RAM mapping - Internal RAM block mapping address.
    • Internal FLASH - If the value of this property is 'yes' the internal FLASH is mapped into the internal address space and has priority over the external address space.
      The following items are available only if the group is enabled (the value is "yes"):
      • Half memory only - If the value of this property is 'yes' the Flash EEPROM is mapped only in second half of memory map.
    • Interrupt vector table - This group contains settings for the placement of interrupt vectors table.
      • Address - Address of the interrupt vector table (IVT). IVT will be placed on this address. This property can be used for bootloader or monitor support.
        The default address is 0xFF80.
      • Size - Size of the interrupt vector table. (For information only)
  • Operating mode and external bus settings - CPU operating mode and external bus settings.
    • Boot operating mode - This property must be set according to the CPU operating mode that is set after reset (respectively before execution of an application). This MCU mode is set by PE5/MODA, PE6/MODB and BKGD/MODC pins during reset (respectively set by debugger).
      Note:
      Processor Expert needs to know an operating mode to allocate IO pins required for selected mode and make some settings available.

      There are 8 options:
      • Special Single Chip: This mode is generally used for debugging single-chip operation, boot-strapping, or security related operations. The active background mode is in control of CPU execution and BDM firmware is waiting for additional serial commands through the BKGD pin. There is no external expansion bus after reset in this mode.
      • Emulation Expanded Narrow: Developers use this mode for emulation systems in which the users target application is Normal Expanded Narrow Mode.
      • Special Test (Expanded Wide): This mode is generally used for debugging single-chip operation, boot-strapping, or security related operations. The active background mode is in control of CPU execution and BDM firmware is waiting for additional serial commands through the BKGD pin. There is no external expansion bus after reset in this mode.
      • Emulation Expanded Wide: Developers use this mode for emulation systems in which the users target application is Normal Expanded Wide Mode.
      • Normal Single Chip: There is no external expansion bus in this mode. The program is executed from internal memory. Ports A, B, K, and most of E are available as general-purpose I/O.
      • Normal Expanded Narrow: Ports A and B are configured as a 16-bit address bus and Port A is multiplexed with 8-bit data. Port E provides bus control and status signals. This mode allows 8-bit external memory and peripheral devices to be interfaced to the system.
      • Peripheral: This mode is intended for Motorola factory testing of the system. The CPU is inactive and an external (tester) bus master drives address, data, and bus control signals.
      • Normal Expanded Wide: Ports A and B are configured as a 16-bit multiplexed address and data bus and Port E provides bus control and status signals. This mode allows 16-bit external memory and peripheral devices to be interfaced to the system.

    • Operating mode switching - This property enables to generate an operating mode switching code, i.e. it enables changing of the boot operating mode by application code. This operating mode is set by the Target operation mode property (available only if this property is set to 'yes') Note: Sometimes it is required to boot into some operation mode and switch to another mode by writing to the MODE register. This property allows user to switch from the boot operating mode into the selected operation mode. If the code for switching of operating mode is undesirable, be aware that most debugging tools provide the ability to self configure the device in the debugging environment (if the part is booted into special signal-chip mode). If the CPU is to be configured for expanded operation in secure mode, the CPU must exit reset in the expanded mode. No writes to the MOD bits are allowed while operating in the secure mode. However, to release security, special single-chip mode must be possible. For more information see the Application note AN2287

There are no items in this mode.
      • Emulation Expanded Narrow - Developers use this mode for emulation systems in which the users target application is Normal Expanded Narrow Mode. The following items are displayed in this mode:
        • Internal Visibility - This property determines whether internal accesses generate a bus cycle that is visible on the external bus.
          If the value of this property is 'Enabled' the internal bus operations are visible on the external bus.
          If the value of this property is 'Disabled' there is no visibility of the internal bus operations on the external bus.
        • External paging - This property must be set to "Enable" in order to enable external paging. If the value of this property is 'Enabled' then if in any expanded mode, PORTK and DDRK are removed from the memory map and port K can be used for external paging. If the value of this property is 'Disabled' then PORTK and DDRK are in the memory map so Port K can be used for general-purpose I/O. In case the port K is not available on the CPU package, the External paging must be either disabled (if possible) or selected operation mode cannot be used (see property Boot operating mode or Target operation mode).
        • Emulate port E - Emulate Port
          If the value of this property is 'Enabled' then if in any expanded mode or special peripheral mode, PORTE and DDRE are removed from the memory map. Removing the registers from the map allows the user to emulate the function of these registers externally.
          If the value of this property is 'Disabled' PORTE and DDRE are in the memory map so Port E can be used for general-purpose I/O.
          In the single-chip mode, PORTE and DDRE are always in the map regardless of the state of this bit.
        • Address/Data signals - Pins associated with the external bus.
          • XABxx - This group contains pins used for external paging. These output pins are enabled if the external paging is enabled (see property External paging). These pins are used to determine which FLASH/ROM or external memory array page is being accessed. At least one pin is required for external paging. If the CPU has no XAADR pin, external paging must be disabled (see property External paging) or selected operation mode cannot be used (see property Boot operating mode or Target operation mode).
