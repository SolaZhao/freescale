Component WatchDog
WatchDog component
Component Level: Low
Category: CPU Internal Peripherals-Timer
Properties:
(Properties are parameters of the component. Please see the Embedded Components page for more information.)
  • Component name - Name of the component.
  • WatchDog device - Watchdog device.
  • WatchDog action - The type of action generated by the watchdog.
  • Output pin - Watchdog output pin (depends on watchdog action).
    The following items are available only if the group is enabled (the value is "Enabled"):
    • Pin - Output pin name.
    • Output pin signal - Signal name of output pin.
  • Period - Watchdog period. It is necessary to enter both a value and an unit (see Timing Setting Syntax). The setting may be made with the help of the Timing dialog box that opens when clicking on (...)button.
  • Settings supported for Freescale HCS12, HCS12X and HCS12Z derivatives only.
    • Settings only if COP support disable ACLK in stop mode
      • COP stops in stop mode - This property determines a behavior of the COP module in the stop mode. If this property is set to 'yes', COP is disabled in stop mode. Hence the COP is static while in stop mode and continues to operate after exit from stop mode. This property has effect only when ACLK is selected as Clock source.
        This item modifies the CSAD bit in the CPMUCLKS register.
    • COP enable during Pseudo-stop mode - If this property is "yes", COP counter will be enabled in the Pseudo-Stop mode.
  • Settings supported for Freescale ColdFireV1, HCS08, HCS12, HCS12X and HCS12Z derivatives only.
    • Mode - Mode of operation.
      When 'Window mode' is selected the Clear method must be used in the last 25% of the selected period. In 'Normal mode' method Clear can be used any time. Window mode selection is available only if the peripheral module supports this feature.
      The following items are available only if the group is enabled (the value is "Window"):
      • Window [%] - This item specifies the value of window in percent of 'Period'.
  • Initialization - Initial settings (after power-on or reset).
    • Enabled in init. code - The component is enabled after a power-on or reset (in initialization code).
  • CPU clock/speed selection - Settings for the CPU speed modes: specifies whether the component is supported or not.
    The following items are available only if the group is enabled (the value is "Supported"):
    • High speed mode - The component is enabled/disabled in high speed mode.
    • Low speed mode - The component is enabled/disabled in low speed mode.
    • Slow speed mode - The component is enabled/disabled in slow speed mode.
