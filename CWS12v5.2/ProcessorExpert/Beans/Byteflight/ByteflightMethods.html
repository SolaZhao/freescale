Component Byteflight
Byteflight communication interface
Component Level: High
Category: CPU Internal Peripherals-Communication
Methods:
(Methods are user-callable functions/subroutines intended for the component functions control. Please see the Embedded Components page for more information.)
  • Enable - Enable the component - module exits initialization mode and starts to connect to the bus. Events may be generated ("DisableEvent"/"EnableEvent").

      ANSIC prototype: byte Enable(void)
    • Return value:byte - Error code, possible codes:
      ERR_OK - OK
      ERR_SPEED - This device does not work in the active speed mode

  • Disable - Disable the component - it stops the send and receive and enters initialization mode. No events will be generated.

      ANSIC prototype: byte Disable(void)
    • Return value:byte - Error code, possible codes:
      ERR_OK - OK
      ERR_SPEED - This device does not work in the active speed mode

  • EnableEvent - Enable the events. This method is available if the interrupt service/event property is enabled and at least one event is enabled.

      ANSIC prototype: byte EnableEvent(void)
    • Return value:byte - Error code, possible codes:
      ERR_OK - OK
      ERR_SPEED - This device does not work in the active speed mode

  • DisableEvent - Disable the events. This method is available if the interrupt service/event property is enabled and at least one event is enabled.

      ANSIC prototype: byte DisableEvent(void)
    • Return value:byte - Error code, possible codes:
      ERR_OK - OK
      ERR_SPEED - This device does not work in the active speed mode

  • SendFrame - The method sends the data through specified transmit buffer and width specified identifier. The method is available only if Transmit buffers are enabled.

      ANSIC prototype: byte SendFrame(byte BufferNum, byte MessageID, byte Length, byte *Data)
    • BufferNum:byte - Number of the buffer to be accessed.
    • MessageID:byte - Identifier of the message.
    • Length:byte - Number of bytes passed as Data. One frame can hold up to 12 bytes of data.
    • Data: Pointer to byte - Pointer to a Data buffer.
    • Return value:byte - Error code, possible codes:
      ERR_OK - OK
      ERR_SPEED - This device does not work in the active speed mode
      ERR_DISABLED - This component is disabled by user
      ERR_RANGE - Some parameter is out of possible range
      ERR_TXFULL - Transmition buffer is full.
      ERR_NOTAVAIL - Transmission buffer is already locked
      ERR_BUSY - The buffer is in use, it cannot be locked at the moment.

  • SendFrameExt - The method retransmits the specified transmit buffer width content previously stored by SendFrame method. The method is available only if Transmit buffers are enabled.

      ANSIC prototype: byte SendFrameExt(byte BufferNum)
    • BufferNum:byte - Number of the buffer to be retransmitted.
    • Return value:byte - Error code, possible codes:
      ERR_OK - OK
      ERR_SPEED - This device does not work in the active speed mode
      ERR_DISABLED - This component is disabled by user
      ERR_RANGE - Some parameter is out of possible range
      ERR_NOTAVAIL - Transmission buffer is already locked
      ERR_TXFULL - Transmition buffer is full.
      ERR_BUSY - The buffer is in use, it cannot be locked at the moment.

  • ReadFrame - The method reads Data from specified receive buffer. If receive FIFO is enabled, reading from buffer 0 reads the message from FIFO. The method is available only if Dedicated receive or receive FIFO buffers are enabled.

      ANSIC prototype: byte ReadFrame(byte BufferNum, byte *MessageID, byte *Length, byte *Data)
    • BufferNum:byte - Number of the buffer to be read.
    • MessageID: Pointer to byte - Pointer to the message indentifier (ID).
    • Length: Pointer to byte - Pointer to length of the received message. The length of the message is up to 12 bytes.
    • Data: Pointer to byte - Pointer to the Data buffer.
    • Return value:byte - Error code, possible codes:
      ERR_OK - OK
      ERR_SPEED - This device does not work in the active speed mode
      ERR_DISABLED - This component is disabled by user
      ERR_RANGE - Parameter BufferNum is out of possible range
      ERR_RXEMPTY - The receive buffer is empty
      ERR_NOTAVAIL - The receive buffer is already locked
      ERR_BUSY - The buffer is in use, it cannot be locked at the moment.

  • SetAcceptanceCode - The method writes acceptance identifier to specified dedicated receive buffer. The Byteflight transfers received message to the receive buffer with matching identifier. The method can be called only if the component is disabled (Disable method).
    The method is available only if Dedicated receive buffers are enabled.

      ANSIC prototype: byte SetAcceptanceCode(byte BufferNum, byte MessageID)
    • BufferNum:byte - Number of the buffer to be accessed.
    • MessageID:byte - Identifier of the message, which will be accepted by the buffer.
    • Return value:byte - Error code, possible codes:
      ERR_OK - OK
      ERR_RANGE - Parameter BufferNum is out of possible range
      ERR_SPEED - This device does not work in the active speed mode
      ERR_ENABLED - This component is enabled by user.
      ERR_NOTAVAIL - The receive buffer is already locked
      ERR_BUSY - The buffer is in use, it cannot be locked at the moment.

  • GetError - The method returns a set of accumulated errors. After calling GetError the accumulated errors is cleared.

      ANSIC prototype: byte GetError(ComponentName_TError *Err)
    • Err: Pointer to ComponentName_TError - Pointer to returned set of errors.
    • Return value:byte - Error code, possible codes:
      ERR_OK - OK
      ERR_SPEED - This device does not work in the active speed mode

  • GetStatus - The method returns a set of accumulated status flags. After calling GetStatus the accumulated status flags are cleared.

      ANSIC prototype: byte GetStatus(ComponentName_TStatus *Status)
    • Status: Pointer to ComponentName_TStatus - Pointer to returned set of status flags.
    • Return value:byte - Error code, possible codes:
      ERR_OK - OK
      ERR_SPEED - This device does not work in the active speed mode

  • GetBufState - The method returns interrupt status flag of the buffer control register. If the buffer is configured as receive, returned value is TRUE if the buffer contains newly received message. If the buffer is configured as transmit, returned value is TRUE if the buffer is empty.

      ANSIC prototype: byte GetBufState(byte BufferNum)
    • BufferNum:byte - no hint
    • Return value:byte -
      true - the flag is set
      false - the flag is cleared or the parameter is out of range.

  • SetAlarm - The method enables alarm sync pulse generation on next cycle. The method is available only in Master mode.

      ANSIC prototype: void SetAlarm(void)

  • ClrAlarm - The method disables alarm sync pulse generation and normal sync pulses starts on next cycle. The method is available only in Master mode.

      ANSIC prototype: void ClrAlarm(void)

  • SetSleepRequest - The method sets the request to switch the module to sleep mode. The module remains stopped until wakeup occurs. GetSleepAck method can be used to test if the module has entered the sleep mode.

      ANSIC prototype: void SetSleepRequest(void)

  • ClrSleepRequest - The method clears the request to switch the module to sleep mode and requests the normal mode entering. GetSleepAck method can be used to test the mode of the module.

      ANSIC prototype: void ClrSleepRequest(void)

  • GetSleepAck - The method tests if the module is in Sleep or Normal run mode.

      ANSIC prototype: bool GetSleepAck(void)
    • Return value:bool -
      true - Module is in Sleep mode.
      false - Module is in Normal run mode.

  • GetInitAck - The method tests if the module is in Initialization (disabled) or Normal run (enabled) mode.

      ANSIC prototype: bool GetInitAck(void)
    • Return value:bool -
      true - The module is in Initialization mode.
      false - The module is in Normal run mode.

  • GetModuleVersion - The method returns version number of the module implementation.

      ANSIC prototype: byte GetModuleVersion(void)
    • Return value:byte - Version number.

  • AbortMessage - The method requests that specified full transmit buffer is aborted. The acknowledge of message abort can be tested by GetAbortMessageAck. AbortMessage method is available only if Transmit buffers are enabled.

      ANSIC prototype: byte AbortMessage(byte BufferNum)
    • BufferNum:byte - Number of buffer to be accessed.
    • Return value:byte - Error code, possible codes:
      ERR_OK - OK
      ERR_RANGE - Parameter BufferNum is out of possible range
      ERR_SPEED - This device does not work in the active speed mode
      ERR_DISABLED - This component is disabled by user

  • GetAbortMessageAck - The method tests if a message in the specified transmit buffer has been aborted (AbortMessage method). GetAbortMessageAck method is available only if Transmit buffers are enabled.

      ANSIC prototype: bool GetAbortMessageAck(byte BufferNum)
    • BufferNum:byte - Number of buffer to be accessed.
    • Return value:bool -
      true - the message has been aborted.
      false - the message has been sent out successfully.

  • StartWakeupPulses - The method causes to transmit wakeup pulses on the transmit pin. The method can be called only if the component is disabled (Disable method).

      ANSIC prototype: byte StartWakeupPulses(void)
    • Return value:byte - Error code, possible codes:
      ERR_OK - OK
      ERR_SPEED - This device does not work in the active speed mode
      ERR_ENABLED - This component is enabled by user

  • StopWakeupPulses - The method causes to stop transmitting wakeup pulses on the transmit pin. The method can be called only if the component is disabled (Disable method).

      ANSIC prototype: byte StopWakeupPulses(void)
    • Return value:byte - Error code, possible codes:
      ERR_OK - OK
      ERR_SPEED - This device does not work in the active speed mode
      ERR_ENABLED - This component is enabled by user


Note: Some methods can be implemented as macros.
