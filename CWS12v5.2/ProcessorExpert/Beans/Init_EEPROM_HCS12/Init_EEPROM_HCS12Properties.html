Component Init_EEPROM for HCS12
EEPROM Memory (EEPROM)
Component Level: Peripheral Initialization
Category: CPU Internal Peripherals-Peripheral Initialization
Properties:
(Properties are parameters of the component. Please see the Embedded Components page for more information.)
  • Component name - Name of the component.
  • Device - Selection of the EEPROM memory device module.
  • Settings - Common EEPROM module settings.
    • Clock settings - Clock/timing settings of the module.
      • Prescaler by 8 - Enables a prescaler by 8, to divide the EEPROM module input oscillator clock before feeding into the CLKDIV divider.
        This item modifies PRDIV8 bit in ECLKDIV register.
      • Clock Divider - This item should divide the EEPROM module input oscillator clock down to a frequency of 150kHz - 200kHz.
        This item modifies EDIV0 to EDIV5 bits in ECLKDIV register.
      • Frequency - Frequency information. This item is influenced by the following properties: Prescaler by 8, Clock Divider.
    • Open EEPROM for program or erase. - Open the EEPROM for program or erase.
      This item modifies EPOPEN bit in EPROT register.
    • Protection of address range - This item determines whether (and how large) there is a protected area in the space of the EEPROM address map.
      This item modifies the EPDIS, EP0, EP1, EP2 bits in EPROT register.
  • Interrupts - Interrupts settings of the module.
    • Command status - Command status interrupt settings.
      • Interrupt - Interrupt vector (for information only).
      • Command complete interrupt - Command complete Interrupt Enable.
        This item modifies CCIE bit in ECNFG register.
      • Command buffer empty interrupt - Command Buffer Empty Interrupt Enable.
        This item modifies CBEIE bit in ECNFG register.
      • Interrupt priority - Priority of Command status interrupts.
      • ISR name - Name of the interrupt service routine (ISR) invoked by this interrupt vector.
        Note: The routine must handle all interrupt flags correctly. The routine must be implemented in the user code, it is not generated by Processor Expert. Find external prototypes of the ISRs in the component module header file.
  • Initialization - Initialization options of the module.
    • Call Init method - The Init method of the component may be called after power-on or reset (in initialization code) by PE.
      yes - The Init method is called by PE.
      no - The Init method is not called by PE, it should be called in user's code.
