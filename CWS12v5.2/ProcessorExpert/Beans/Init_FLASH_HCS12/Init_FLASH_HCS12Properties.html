Component Init_FLASH for HCS12
Flash Memory (FLASH)
Component Level: Peripheral Initialization
Category: CPU Internal Peripherals-Peripheral Initialization
Properties:
(Properties are parameters of the component. Please see the Embedded Components page for more information.)
  • Component name - Name of the component.
  • Device - Selection of the Flash memory device module.
  • Settings - Common Flash module settings.
    • Clock settings - Clock/timing settings of the module.
      • Settings only if "IntFLASH" supports "Prescaler by 8" divider.
        • Prescaler by 8 - Enables a prescaler by 8, to divide the Flash module input oscillator clock before feeding into the CLKDIV divider.
          This item modifies the PRDIV8 bit in the FCLKDIV register.
      • Clock Divider - This item should divides the Flash module input oscillator clock down to a frequency 150kHz - 200kHz for HCS12 and HCS12X flash module without Emulated EEPROM (P-Flash) feature, and to a frequency 0.8 MHz - 1.05 MHz for a flash module with Emulated EEPROM (P-Flash) feature.
        This item modifies the FDIVn bits in the FCLKDIV register.
      • Settings only if FLASH module support locking of divider register.
        • Clock divider locked - If Clock divider value is locked, it cannot be changed (it become writtable only after reset). This item modifies FDIVLCK bit in FCLKDIV register.
      • Frequency - Frequency of the Flash internal clock (for information only). This item is influenced by the following properties: Prescaler by 8, Clock Divider.
    • Settings supported for Freescale HCS12 derivatives only.
      • Settings only if the Flash module supports bank switching.
        • Register Bank Select - This item determines which of the available Flash register bank is initialized last in the initialization sequence.
          If the selected cpu contains more than one Flash block, this item modifies BKSEL bit (or BKSEL[1:0] bits).
        • Flash blocks - List of Flash block arrays on the selected derivative.
          One Item of the list looks like:
          Flash Block 0 - Setting of a Flash block
          • Flash Protection Open - The function of this item depends on the specific type of the Flash module (please check the datasheet).
            On older devices, the status of this property opens the Flash array for program or erase.
            • "no" - the whole Flash block is protected and the Protection of Higher Address range and the Protection of Lower Address range properties have no effect.
            • "yes" - the Flash block is not protected and enabled for program or erase.
            On newer devices, the status of this property determines the protection function for program or erase.
            • "no" - the Protection of Higher Address range and the Protection of Lower Address range properties defines area to be unprotected and the rest of the Flash block is protected (useful for EEPROM emulation).
            • "yes" - the Flash block is not protected and enabled for program or erase.
            This item modifies the FPOPEN bit in the FPROT register.
          • Protection of Higher Address range - This item determines whether (and how large) there is a protected area in the higher space of the Flash address map.
            This item modifies the FPHDIS bit and FPHS0 to FPHS1 bits in the FPROT register.
          • Protection of Lower Address range - This item determines whether (and how large) there is a protected area in the lower space of the Flash address map.
            This item modifies the FPLDIS bit and FPLS0 to FPLS1 bits in the FPROT register.
    • Settings supported for Freescale HCS12 G, HY, P, VR and HCS12X derivatives only.
      • Protection settings - Protection settings
        • Flash Protection Open - This property determines the protection function for program or erase.
          • "no" - the Protection of Higher Address range and the Protection of Lower Address range properties defines area to be unprotected and the rest of the flash block is protected (useful for EEPROM emulation).
          • "yes" - the Flash block is not protected and enabled for program or erase.
          This item modifies the FPOPEN bit in the FPROT register.
        • Protection of Higher Address range - This item determines whether (and how large) there is a protected/unprotected area in the higher space of the Flash address map.
          This item modifies the FPHDIS bit and FPHS0 to FPHS1 bits in the FPROT register.
        • Protection of Lower Address range - This item determines whether (and how large) there is a protected/unprotected area in the lower space of the Flash address map.
          This item modifies the FPLDIS bit and FPLS0 to FPLS1 bit in the FPROT register.
        • Settings only if D-Flash supports DFPROT register for protection settings.
          • D-Flash protection control - Enables D-Flash memory protection control with size defined by following property.
            This item modifies the DPOPEN bit in the DFPROT register.
          • Protection size - Size of the protected D-Flash area.
            This item modifies the DPS[3:0] bits in the DFPROT register.
        • Settings only if flash support P-Flash module with EEPROM emulation.
          • Buffer RAM Writes - The item enables writes to the Buffer RAM partitioned for EEE. This item modifies EPOPEN bit in the EPROT register.
          • Buffer RAM protection - The item sets the protection of buffer RAM EEE partition. This item modifies EPDIS and EPS[2:0] bits in EPROT register.
  • Interrupts - Interrupts settings of the module.
    • Command status - Command status interrupt settings.
      • Interrupt - Interrupt vector (for information only).
      • Complete interrupt - Command complete interrupt.
        This item modifies the CCIE bit in the FCNFG register.
      • Settings only if the Flash module supports "Empty interrupt flag"
        • Empty interrupt - Command buffer empty interrupt.
          This item modifies the CBEIE bit in the FCNFG register.
      • Interrupt priority - Priority of Command status interrupt.
      • ISR name - Name of the interrupt service routine (ISR) invoked by this interrupt vector.
        Note: The routine must handle all interrupt flags correctly. The routine must be implemented in the user code, it is not generated by Processor Expert. Find external prototypes of the ISRs in the component module header file.
    • Settings only if the flash supports P-Flash module.
      • Error - Error interrupt settings.
        • Interrupt - Interrupt vector (for information only).
        • Settings only if the flash module supports Emulated EEPROM feature.
          • Erase error interrupt - The item controls interrupt generation when a failure is detected during an EEE erase operation.
            This item modifies ERSERIE bit in the FERCNFG register.
          • Program error interrupt - The item controls interrupt generation when a failure is detected during an EEE program operation.
            This item modifies PGMERIE bit in the FERCNFG register.
          • Protection violation error interrupt - The item controls interrupt generation when a protection violation is detected during a write to the buffer RAM EEE partition.
            This item modifies EPVIOLIE bit in the FERCNFG register.
          • Error Type 1 interrupt - The item controls interrupt generation when a change state error is detected during an EEE operation.
            This item modifies ERSVIE1 bit in the FERCNFG register.
          • Error Type 0 interrupt - The item controls interrupt generation when a sector format error is detected during an EEE operation.
            This item modifies ERSVIE0 bit in the FERCNFG register.
        • Double bit fault detect interrupt - The item controls interrupt generation when a double bit fault is detected during a Flash block read operation.
          This item modifies DFDIE bit in the FERCNFG register.
        • Single bit fault detect interrupt - The item controls interrupt generation when a single bit fault is detected during a Flash block read operation.
          This item modifies SFDIE bit in the FERCNFG register.
        • Ignore single bit fault - If true, single bit faults detected during array reads are not reported and single bit fault will not be generated.
          This item modifies the IGNSF bit in the FCNFG register.
        • Force double bit fault detect - The item allows the user to simulate a double bit fault during flash read operation. If yes, any flash read operation will force "Double bit fault detect interrupt flag" to be set.
          this item modifies the FDFD bit in the FCNFG register.
        • Force single bit fault detect - The item allows the user to simulate a single bit fault during flash read operation. If yes, any flash read operation will force "Single bit fault detect interrupt flag" to be set.
          this item modifies the FSFD bit in the FCNFG register.
        • Interrupt priority - Priority of Error interrupt.
        • ISR name - Name of the interrupt service routine (ISR) invoked by this interrupt vector.
          Note: The routine must handle all interrupt flags correctly. The routine must be implemented in the user code, it is not generated by Processor Expert. Find external prototypes of the ISRs in the component module header file.
    • Settings supported for Freescale HCS12 KC128, KG128, KG256, KL128, KT256 derivatives only.
      • Double fault detect - Double fault detect interrupt settings.
        • Interrupt - Interrupt vector (for information only).
        • Double fault detect - Double fault detect interrupt enable.
          This item modifies the DFDIE bit in the FCNFG register.
        • Interrupt priority - Priority of Double fault interrupt.
        • ISR name - Name of the interrupt service routine (ISR) invoked by this interrupt vector.
          Note: The routine must handle all interrupt flags correctly. The routine must be implemented in the user code, it is not generated by Processor Expert. Find external prototypes of the ISRs in the component module header file.
  • Initialization - Initialization options of the module.
    • Call Init method - The Init method of the component may be called after power-on or reset (in initialization code) by PE.
      yes - The Init method is called by PE.
      no - The Init method is not called by PE, it should be called in the user's code.
