Component IntFLASH
Internal FLASH
Component Level: High
Category: CPU Internal Peripherals-Memory
Component types:
(User types are derived from basic types and they are designed for usage in the driver interface. They are declared in the generated code.)
  • ComponentName_TAddress = dword - The type for the address to the FLASH. Address refers to the smallest addresable unit. Size of this type depends on the selected CPU family. See also page General Info in the component's documentation.
  • ComponentName_TDataAddress = byte* - Address of the data block in memory for methods Get/SetBlockFlash. To maintain backward compatibility, it is double-word on Freescale 56800/E family, otherwise it is pointer to the smallest addressable unit (byte).
Component constants:
(Constants defined by this component for usage in the driver interface. They are defined in the generated code.)
  • ComponentName_AREA_COUNT - Number of flash memory areas. For each area there are several constants with information about the area.
  • ComponentName_AREA_n_START - Flash area start address (in the format used by component methods). "n" is a number of the area (first area is 0 and last is AREA_NUMBER - 1).
  • ComponentName_AREA_n_SIZE - Flash area size. Value is in minimal addressable units (usualy bytes). "n" is a number of the area (first area is 0 and last is AREA_COUNT - 1).
  • ComponentName_AREA_n_END - Flash area last address (in the format used by component methods). It is equal to AREA_n_START + AREA_n_SIZE - 1. "n" is a number of the area (first area is 0 and last is AREA_NUMBER - 1).
  • ComponentName_AREA_n_SECTOR_SIZE - Flash area sector size: minimal erasable unit. Value is in minimal addressable units (usualy bytes). "n" is a number of the area (first area is 0 and last is AREA_NUMBER - 1).
  • ComponentName_AREA_SECTOR_SIZE - Minimal erasable unit. Value is in minimal addressable units (usualy bytes). This constant is defined only if sector size of all areas are the same.
  • ComponentName_PROGRAMMING_PHRASE - Size of programming phrase, i.e. number of bytes that must be programmed at once.
  • ComponentName_ALLOW_CLEAR - TRUE if the component setting allows clearing bits of already programmed flash memory location without destruction of the value in surrounding addresses by sector erase. For 'Write' programming mode, please refer to datasheet if the successive rewrite is recommended by the chip producer.
  • ComponentName_VIRTUAL_PAGE_SIZE - Size of the virtual page in minimal addressable units (usually in bytes). The constant is defined only if the size is non-zero. See also property Virtual Page.
