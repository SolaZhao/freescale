Component Init_RTI for HCS12X
Real Time Interrupt (RTI)
Device Initialization
(Description of the component parameters.)
  • Device - Counter of the RTI.
  • Settings - Common RTI module settings.
    • Clock settings - Clock/timing settings of the module.
      • The following settings is available only if RTI supports clock select
        • Clock source - Select of the RTI clock source.
          This item modifies RTIOSCSEL bit in CPMUCLKS register.
      • Prescaler - This item selects the prescaler rate for the RTI.
        This item modifies RTR4, RTR5 and RTR6 bits in RTICTL/CPMURTI register.
      • Modulus - This item selects the modulus counter target value to provide additional granularity.
        This item modifies RTR0, RTR1, RTR2 and RTR3 bits in RTICTL/CPMURTI register.
      • The following settings is available only if RTI supports divider select
        • Divider - This item selects decimal or binary based prescaler values.
          This item modifies the RTDEC bit in RTICTL/CPMURTI register.
      • Period - Period value . This item is influenced by the following properties: Prescaler, Modulus.
    • Settings only if RTI support "Stop In Wait" feature
      • Stop in Wait Mode - If "yes" RTI stops and initializes the RTI dividers whenever the part goes into Wait Mode.
        This item modifies RTIWAI bit in CLKSEL register.
    • Settings not supported for Freescale HCS12 UF derivatives.
      • RTI enable in Pseudo Stop Mode - RTI enable during Pseudo Stop Mode.
        This item modifies bit PRE in PLLCTL/CPMUCLKS register.
  • Interrupts - Interrupts settings of the module
    • RTI Interrupt - RTI Interrupt settings
      • RTI Interrupt - RTI Interrupt enable.
        This item modifies RTIE bit in {CRG/CPMU}INT register
      • Interrupt - Interrupt vector (for information only)
      • Priority - Priority of RTI interrupt.
      • ISR name - Name of the interrupt service routine (ISR) invoked by this interrupt vector.
        Note: The routine must handle all interrupt flags correctly. The routine must be implemented in the user code, it is not generated by Processor Expert. Find external prototypes of the ISRs in the component module header file.
