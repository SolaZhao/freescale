Component FreescaleEMAC
Configuration modes for MC9S12NE64 EMAC start-up
Component Level: Low
Category: CPU Internal Peripherals-Communication
Properties:
(Properties are parameters of the component. Please see the Embedded Components page for more information.)
  • Component name - Name of the component.
  • Buffer mapping - Buffer Mapping Configuration
  • Memory occupation - Occupation of memory - for information only
    • Buffer start addr - Base address in RAM for RX and TX buffers. For information only. It can be set in the CPU.
    • Buffer size - Total sum of RX and TX buffers - for information only
  • EPHY device - Selected Ethernet Transceiver device.
  • EMAC device - Selected Ethernet Media Access Controller device.
  • MAC address - Unique address of device. Enter as hexadecimal number (12 digits required).
  • Physical address - EPHY address for MII requests
  • Bus clock - Bus clock setting - for information only. This can be set in the cpu.
  • Auto negotiation - Enable auto negotiation
    The following items are available only if the group is enabled (the value is "Enabled"):
    • Half 100 - Advertise Half duplex at speed 100 MBps
    • Full 100 - Advertise Full duplex at speed 100 MBps
    • Half 10 - Advertise Half duplex at speed 10 MBps
    • Full 10 - Advertise Full duplex at speed 10 MBps
  • Manual settings - If Auto negotiation is disabled, set communication speed and mode
    • Speed 100 - Enable 100 MBps. Available if the Auto negotiation is disabled.
    • Full duplex - Enable full duplex. Available if the Auto negotiation is disabled.
  • EMAC control - Setting for EMAC controlling (broadcast, multicast, etc.)
    • RX poll mode - RX mode - poll or interrupt driven (Not implemented into Viola Stack)
    • Delete babbling frames - If enabled, packets larger the maximum frame length (babbling error) will be deleted.
    • RX max. frame length - Receive maximum frame length (Values from range 0-1536)
    • Broadcast reject - All broadcast address frames are rejected.
    • Multicast hash table - Multicast hash table is used for checking multicast addresses.
    • Prom mode - All frames are received regardless of address.
    • Ethertype control - Filtering based on the ethertype.
      • Accept all ethertypes - All frames are received regardless of address.
      • Programmable ethertype - Accept Programmable Ethertype, 'etype' parameter is used.
        The following items are available only if the group is enabled (the value is "yes"):
        • Programmable Ethertype - Programmable Ethertype value (16bit value). Used when Programmable Ethertype is enabled.
      • Emware ethertype - Accept Emware Ethertype. Enabled when the Accept all ethertypes is set to "no"
      • IPV6 version - Accept Internet IP version (IPV6) Ethertype. Enabled when the Accept all ethertypes is set to "no"
      • IPV4 version - Accept Internet IP version (IPV4) Ethertype. Enabled when the Accept all ethertypes is set to "no"
      • ARP ethertype - Accept Address Resolution Protocol (ARP) Ethertype. Enabled when the Accept all ethertypes is set to "no"
      • IEEE802.3 length field ethertype - Accept IEEE802.3 Length Field Ethertype. Enabled when the Accept all ethertypes is set to "no"
    • Flow control - Flow control settings
      • Flow control - Enable flow control in full duplex
      • Pause time - Enter value for PAUSE duration parameter in units of slot times (512 bit times)
  • Optimalizations - Other settings which have influence on effectivity of generated code
    • Zero copy - Zero Copy Mode. Setting this mode not only can conserve RAM but can also increase performance. In this mode, no copy of the receive buffer is created, so the data is processed in the buffer.
      This property can be set in the parent component.
    • Word Access - Word Access Mode. This mode allows word access to buffers instead of byte access to increase overall system performance.
      This property can be set in the parent component.
  • SW LED control - Use user software to drive EPHY status indicators on port L. Otherwise (when disabled) use EPHY hardware LED drive function to drive 5 EPHY status indicators.
    The following items are available only if the group is enabled (the value is "Enabled"):
    • Activity LED - Use software to drive an EPHY activity LED
      The following items are available only if the group is enabled (the value is "yes"):
      • Pin - Pin for activity LED. For information only.
    • Link LED - Use software to drive an EPHY link LED
      The following items are available only if the group is enabled (the value is "yes"):
      • Pin - Pin for link LED. For information only.
    • Speed LED - Use software to drive an EPHY speed LED
      The following items are available only if the group is enabled (the value is "yes"):
      • Pin - Pin for link LED. For information only.
    • Duplex LED - Use software to drive an EPHY duplex LED
      The following items are available only if the group is enabled (the value is "yes"):
      • Pin - Pin for link LED. For information only.
    • Collision LED - Use software to drive an EPHY collision LED
      The following items are available only if the group is enabled (the value is "yes"):
      • Pin - Pin for link LED. For information only.
